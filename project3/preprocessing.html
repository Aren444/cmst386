CSS Preprocessing with LESS

Client Side vs Server Side

Client side preprocessing means the browser downloads the .less file and converts it to CSS using JavaScript. That is helpful while testing styles but it can slow down loading because the browser has extra work to do. Server side preprocessing uses a tool like Node or an online compiler to convert .less to regular .css before uploading a site. That is faster and is usually used for real websites.

LESS Features

LESS has lots of tools that make your CSS more organized. Variables let me store colors and spacing values in one place so I can reuse them instead of re-typing out a layout again. I like to think of mixins like my kitchen recipes because I can create a group ingredient types. My dry items like flour,salt, baking soda go in one bowl, So for buttons, for example, could have the "bowl" of button styling "ingredients", and then apply it to other things when applicable. Lighten and darken functions are nice also to adjust to background colors appropriately. It's usefull all around, but almost a necessity for big websites to stay consistent.
